Definition

FLuaG (Floating Lua Graphics) is an Avisynth plugin for video/audio data editing via Lua scripting.

Allowing to transfer image/sample data between Avisynth and Lua, it works like a filter SDK. Inside Lua scripts it's possible to write 3D effects with OpenGL or others things by importing libraries to the Lua API.
Regrettably FLuaG slows the process much more than an original filter in C/C++ but it makes the work easier.

Features

  • All in one file
  • Video & audio access
  • OpenGL 2.1 + GL_ARB_imaging + GL_ARB_framebuffer_object support
  • Text extents & outline path
  • PNG image loading & saving
  • Math geometry functions
  • Onscreen rendering

Plugin

FLuaG is a C++ plugin, so you have to load it simply with LoadPlugin.
LoadPlugin("SYSTEM_PATH\FLuaG.dll")
Now you can call function FLuaG, defined as...
FLuaG(clip c, string video = void, string audio = void, bool onscreen = false, bool samples32 = false, string userdata = void)
clip c: Video and/or audio clip. Needed!
string video: Lua script for editing video data. Optional.
string audio: Lua script for editing audio data. Optional.
bool onscreen: Window for OGL rendering visible? Optional.
bool samples32: Samples with size of 32 bit instead of 16 bit? Optional.
string userdata: String containing avs variable names for usage. Optional.

Audio

For audio editing, a Lua script has to be send to FLuaG for this job.
It will be called at Avisynth startup. If there's a GetSamples function defined, this function will be called with index and time in milliseconds of current start sample and number of samples this function run during playback.
function GetSamples(start_sampe_index, start_sample_ms, count)

end
For loading current samples, call LoadSamples(). It returns a table with 16bit or 32bit samples, depending on FLuaG parameter samples32.
For saving new data to current samples, call SaveSampes(samples). samples should be a table with same size like loadable samples table and valid numbers.
Meta data about audio stream are stored in global variables. There're AUDIO_CHANNELS (number of audio channels), AUDIO_SAMPLE_RATE (frequency = samples/second per channel) and AUDIO_SAMPLES (number of samples per channel).

Video

For video editing, a Lua script has to be send to FLuaG for this job.
It will be called at Avisynth startup. If there's a GetFrame function defined, this function will be called with index and time in milliseconds of current frame during playback.
function GetFrame(frame_index, frame_ms)

end
For loading current frame data, call LoadFrame(). It returns a table with RGBA data.
For loading current frame data directly to OGL context, call LoadFrameToContext().
For saving new data to current frame, call SaveFrame(frame). frame should be a table with same size like loadable frame and valid RGBA data.
For saving new data to current frame from OGL context, call SaveFrameFromContext().
Meta data about video stream are stored in global variables. There're VIDEO_WIDTH (width of frames in pixels), VIDEO_HEIGHT (height of frames in pixels), VIDEO_FPS (number of frames per second) and VIDEO_FRAMES (number of frames).

Examples

test.avs
BlankClip(length=250, width=704, height=396, pixel_type="RGB32", fps=25, audio_rate=48000, channels=2, sample_type="16bit")

point_size = 50
FLuaG(audio="test_audio.lua", video="test_video.lua", onscreen=false, samples32=false, userdata="point_size")

test_audio.lua
function GetSamples(start_sample_index, start_sample_ms, count)
    local samples = {}

    for i=1, count * AUDIO_CHANNELS do
        samples[i] = math.random(-32768, 32767)
    end

    SaveSamples(samples)
end

test_video.lua
glPointSize(USERDATA[1])

function GetFrame(frame_index, frame_ms)
    LoadFrameToContext()

    glBegin(GL_POINTS)
        glVertex(0,0)
    glEnd()

    SaveFrameFromContext()
end
test.avs: using FLuaG on a blank clip to edit audio data by "test_audio.lua" and video data by "test_video.lua"
test_audio.lua: random noise
test_video.lua: point at mid

For more examples, look into the "examples" folder from FLuaG download.

Calculates point x|y with given angle on outline of ellipse with center point mx|my and given width and height.
num = math.interpolate(a, b, pct)
Interpolates number a to number b by factor pct.
hit = math.intriangle(px, py, pz, x1, y1, z1, x2, y2, z2, x3, y3, z3)
Tests point px|py|pz occurence in triangle with edge points x1|y1|z1, x2|y2|z2 and x3|y3|z3.
len = math.length(x, y, z)
Calculates length of vector in x, y and z direction.
... = math.normalize(...)
Normalizes given numbers.
num = math.randomsteps(start, end, step)
Returns random number num between start and end with offsets of step.
rx, ry, rz = math.reflect(x, y, z, nx, ny, nz)
Calculates reflection vector in rx, ry and rz direction of ray vector in x, y and z direction, reflected on plain with normal vector in nx, ny and nz direction.
int = math.round(num)
Rounds number num to integer.

If num is less than 0, then n is -1. If num is greater than 0, then n is 1. If num is equal to 0, then n is 0.

Media access

Here are all functions and variables listed to work with audio/video streams.

Audio

Attributes 

AUDIO_CHANNELS
AUDIO_SAMPLE_RATE
AUDIO_SAMPLES

Functions

samples = LoadSamples()
Returns table samples with 16/32 bit numbers. These are the samples at the current position in audio stream. Their start index you get from the GetSamples function call. The number of samples differs from file to file.
SaveSamples(samples)
Saves table samples at current audio stream position. Number of table elements have to be equal to the number of elements from table you get from LoadSamples(). Table elements should be 16/32 bit numbers.

Video

Attributes 

VIDEO_WIDTH
VIDEO_HEIGHT
VIDEO_FPS
VIDEO_FRAMES

Functions

frame = LoadFrame()
Returns table frame with RGBA data. This is the frame at the current position in video stream. His index you get from the GetFrame function call. Table size = VIDEO_WIDTH * VIDEO_HEIGHT * 4.
SaveFrame(frame)
Saves table frame at current video stream position. Number of table elements have to be equal to VIDEO_WIDTH * VIDEO_HEIGHT * 4. Table elements have to be RGBA data, so they should be 8 bit numbers.
LoadFrameToContext()
Transfers RGBA data from the frame at the current position in video stream to OGL context. Frame index you get from the GetFrame function call.
SaveFrameFromContext()
Transfers RGBA data from OGL context to the frame at the current position in video stream.

Returns table samples with 16 bit samples.

Returns table frame with RGBA data.

gluDrawPath(path)
Tesselator as path system.
path is a table with path points in following format:
.typePoint type. Should be GLU_MOVE, GLU_LINE or GLU_ENDLINE.
.xPoint position horizontal in pixels.
.yPoint position vertical in pixels.
.zPoint position in depth.
.rPoint red color channel. Optional.
.gPoint green color channel. Optional.
.bPoint blue color channel. Optional.
.aPoint alpha color channel. Optional.
Other elements beside points are following:
.nxHorizontal direction of object normal. Optional.
.nyVertical direction of object normal. Optional.
.nzDepth direction of object normal. Optional.
.windingruleTesselation winding rule. Optional.

FAQ

Why OpenGL and not Direct3D?
I don't want to discuss the benefits of OGL and D3D, but one point was the thought of making it platform independent.

Why Lua? Why not Python/Ruby/...?
Lua is faster, smaller and easier to learn. Accept it.

Why i cannot compile/execute it?
Read the error message. Maybe you forgot to fix file paths.

How do i realize *** ?
Look into the examples you got or learn Lua/OpenGL (see Links).

How i can use more than one clip? / How i can use audio data for drawing?
Send your clip as userdata to the filter process. It becomes a Lua object with access to video and audio data.

How i can speed up my filters?
Look for OpenGL performance tips and minimize code in GetFrame/GetSamples function definition.

How i should work with FLuaG?
It's recommend to work in 2 windows:
-AvsP for preview + Avisynth script edit.
-Any Lua editor like Scite or Notepad++.

I miss function *** . Could i get it?
Maybe. Ask!
